Quality: About This Data Set

The data used in this byte is about WiFi connections that my iPhone has accessed in the past week or so. The data was collected via the AWARE client downloaded via the App Store.

Is the data complete?

After exporting and examining the data set from the WiFi table collected by AWARE, there were some entries that did not have an SSID and BSSID associated with it. This is due to not being connected to WiFi at the time of collection. For the purpose of the visualization, I excluded these entries because I wanted to visualize in particular the WiFi connections. Other than these missing entries, every column for entries associated with a WiFi connection had non-empty values.

Is the data coherent?

The data is pretty coherent. There were three main columns that I used: the SSID, which is a "human-readable" name of WiFi network, the BSSID, which is the MAC address of the connected network, and the timestamp, which was the UNIX timestamp at which the network was connected to. For more information about the sensed fields, you can check out the documentation here.

In terms of preprocessing the WiFi table, the only field that really needed any preprocessing was the timestamp, which I converted to a human-readable date format. The other fields were simply the network names and MAC addresses, and are inherently logical.

Is the data correct?

To be using a third-party client to be collecting information from my phone, I can't be 100% sure that the data is correctly sensed and recorded. From what I can tell, the AWARE client is pretty accurate. There were some entries that didn't make sense (being connected to multiple SSID's at once), so I excluded the overlapping entries at a specific point of time. Note that it does make sense to be connected to multiple BSSID's, because a network could have different access points. Unexpectedly, there are many WiFi devices that connect to "CMU-SECURE", as compared to my room "tree o four", which only has a few access points.
